Part 1: Autofill

One weird issue I faced had to do with autofilling of the contact form. When the browser autofills, it applies a style called input:-internal-autofill-selected. More specifically, it looks like this: 

          
input:-internal-autofill-selected {
  appearance: menulist-button;
  background-image: none !important;
  background-color: -internal-light-dark(rgb(232, 240, 254), rgba(70, 90, 126, 0.4)) !important;
  color: fieldtext !important;
}

It makes the input field change color from the normal dark gray background and light gray text to black text with white background. (If you're wondering what I mean with dark gray background and light gray text, then you're probably on light mode. I probably should've mentioned that I work mostly on dark mode.)

Now, while it is perhaps helpful in that it indicates how the browser stores the autofill data in itself, the form I have doesn't have any particularly sensitive information (debatably the email address), I think I prefer not suddenly blinding users.

One thing to note is that these stylings use the !important tag, which indicates that they shouldn't be overwritten by other CSS. In order to get around that (kinda hacky, not great), I instead chose to use the following: 

            
              input:-webkit-autofill,
              input:-webkit-autofill:focus {
                  transition: background-color 600000s 0s, color 600000s 0s;
              }

Basically, what it does is force the background color change from dark to light occur over 600000s, or 1000 minutes, which, realistically, should never finish.

Part 2: Custom Dialogs

Newly added to the sidebar are the dialog and custom dialog sections - you'll notice there are two custom dialog sections; I'll get to that soon, don't worry. If you didn't already know, dialogs are basically a way in which you can use JavaScript in order to get input from the users, whether it's textual, date, boolean, etc.

You can compare the native JavaScript dialogs and the ones I created in the links on the sidebar. Now, you may be asking, why custom dialogs? Are the native ones not good enough for you? Well, kind of? HTML (not JavaScript!) has the (customizable) native <dialog> tag, but as part of the assignment, we weren't supposed to use that tag. Therefore if we want a customizable dialog, we had to make our own (or import it, but that's cheating.)

Note: As it turns out, we were supposed to use <dialog>, so I simply made my own life harder with my lack of reading ability, as per usual. Well, I suppose that's a lesson learned. Reading is important! Regardless, the content below is still worth talking about, I think. Note that the second custom dialogs section in the sidebar is using dialogs; I reworked my code (again, as you'll see) in order to implement it.

My initial attempt at creating a dialog was have a base structure with all of the necessary elements (buttons, inputs, etc.), and I would just show/hide them as necessary depending on the type of dialog I needed. At first glance, that seemed to work well enough, and you can see that in the first custom dialogs link in the sidebar.

However, as I began the subsequent part of the assignment, implementing a simple CRUD blog (not this one, the other one), I found that I would require multiple different inputs, which would require a different dialog structure than the expected on from my original implementation, so it wouldn't work proper without some serious retooling. In lieu of that, I decided to rework the dialogs so I could slot in different inputs as necessary for different dialogs.

Rithmatics

One of my favorite books is The Rithmatist, by Brandon Sanderson. In particular, the system of Rithmatics is fascinating to me. Rithmatics is a magic system that uses chalk. Drawing different things has different effects, some of which are detailed below.

Lines
Type Description Example
Line of Warding
  • Round shape (circle or ellipse)
  • Strength is proportional to curve
  • Require anchoring at bind points
Line of Forbiddance
  • Straight line
  • Strength is proportional to thickness
  • Creates impenetrable barrier perpendicular to the surface (includes physical objects)
  • Spoilers approaching: A Line of Forbiddance was shown to even be able to stop... A bullet, and later mentioned to block cannonballs.
Line of Vigor
  • Squiggly line (sine wave)
  • Strength is proportional to curve (more curves = stronger)
  • Reflect off Lines of Forbiddance
Line of Making
  • Variable shape
  • Strength varies with complexity and aesthetic
  • Used to create chalklings
  • Can only interact with chalk without Glyph of Rending

One particular interest is how exactly aesthetic beauty, which determines the strength of chalklings, is determined. Is there actually some form of objective measure, as assumed by the characters in the story?

While it's true that some things are clearly less well drawn than others (see: my poor attempt at an icon (if you're reading this, it's coming soon™)), I think there exists a point at which objectivity bleeds into subjectivity.

For example, the Mona Lisa is considered one of the best painintgs in the modern world—if it were converted to a chalkling (somehow), how strong would that be? Could we create a scale from my poor attempt at drawing a logo to the Mona List in terms of chalkling strength? Food for thought.
